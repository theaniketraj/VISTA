# Configuration

VISTA is designed to work out of the box with reasonable defaults, but it is also highly configurable.

## Plugin Extension

Configure the plugin in your `top-level` or module-level `build.gradle.kts` file using the `vista` extension block.

```kotlin
vista {
    // Defines the name of the file where version properties are stored.
    // Default: "version.properties"
    versionFileName.set("version.properties")
}
```

## Property File Structure

The properties file (default `version.properties`) must contain the following keys:

```properties
VERSION_MAJOR=1
VERSION_MINOR=0
VERSION_PATCH=0
BUILD_NUMBER=0
```

- **VERSION_MAJOR**: Significant changes, breaking changes.
- **VERSION_MINOR**: New features, non-breaking.
- **VERSION_PATCH**: Bug fixes, non-breaking.
- **BUILD_NUMBER**: CI/CD run number or incremental build counter.

## Environment Variables

All properties can be overridden at runtime using environment variables. This is particularly useful for CI/CD pipelines where the build number is generated by the CI server (e.g., GitHub Actions, Jenkins).

| Environment Variable | Target Property |
| :--- | :--- |
| `VISTA_VERSION_MAJOR` | `VERSION_MAJOR` |
| `VISTA_VERSION_MINOR` | `VERSION_MINOR` |
| `VISTA_VERSION_PATCH` | `VERSION_PATCH` |
| `VISTA_BUILD_NUMBER` | `BUILD_NUMBER` |

**Precedence Order:**

1. Environment Variable (Highest)
2. Property File Value
3. Default (0) (Lowest)
